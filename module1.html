AI vs ML vs GenAI Landscape

📖 12 min read 📚 Topic 1 of 13
Why This Matters

Before building AI agents, you need to understand what AI actually is — and isn't. These terms get thrown around interchangeably, but they mean very different things. Understanding the landscape helps you choose the right tool for the right job.

What is Artificial Intelligence?

Artificial Intelligence (AI) is a broad field of computer science focused on creating machines that can perform tasks that typically require human intelligence. Think of it as an umbrella — everything we discuss in this course falls under it.

AI includes things like playing chess, recognizing faces in photos, translating languages, and driving cars. Some AI is simple (a thermostat adjusting temperature) and some is incredibly complex (ChatGPT writing essays).

What is Machine Learning?

Machine Learning (ML) is a subset of AI. Instead of programming explicit rules ("if temperature > 30, turn on AC"), you give the computer lots of examples and let it learn the rules on its own.

Think of it like teaching a child to recognize dogs. You don't explain "four legs, fur, tail" — you show them hundreds of pictures of dogs until they just know. That's ML in a nutshell.

What is Deep Learning?

Deep Learning is a subset of ML that uses neural networks — layered mathematical structures inspired (very loosely) by the brain. "Deep" means many layers. These networks excel at finding patterns in complex data like images, audio, and text.

What is Generative AI?

Generative AI (GenAI) is a subset of Deep Learning focused on creating new content — text, images, code, music. ChatGPT, DALL-E, and Midjourney are all GenAI. Unlike traditional ML that classifies or predicts, GenAI generates.

Key Insight

All GenAI is AI, but not all AI is GenAI. A spam filter uses ML (classification). ChatGPT uses GenAI (generation). Both are AI.

VS Code: Your Code Editor

Visual Studio Code (VS Code) is a free, powerful code editor made by Microsoft. Think of it as a smart notepad — it understands code, highlights syntax in colors, and catches errors as you type.

Google Colab: Code in the Cloud

Google Colab is like Google Docs, but for code. You write Python in your browser, and Google's servers run it. No installation needed — perfect for beginners.

  1. Go to colab.research.google.com
  2. Sign in with your Google account
  3. Click "New Notebook"
  4. Type print("Hello World") in a cell and press Shift+Enter
Pro Tip

Colab gives you free GPU access! Go to Runtime → Change runtime type → GPU. Useful for ML later.

Managing Secrets & API Keys

API keys are like passwords for services (OpenAI, Google, etc.). Never put them directly in your code!

  • .env files: Store keys in a file called .env, load them with python-dotenv
  • Environment variables: Set them in your terminal so your code can read them
  • Colab Secrets: Use the 🔑 icon in Colab's sidebar
Security Rule

NEVER commit API keys to GitHub. Add .env to your .gitignore file. If you accidentally push a key, revoke it immediately.

Creating a GitHub Account

Conditionals & Boolean Logic

📖 12 min read🔀 Topic 5 of 13
Why This Matters

Conditionals let your program make decisions — "if this, then do that." AI agents constantly make decisions: if the user asked about weather, call the weather tool; if they asked about code, generate code. Conditionals are the foundation of all decision-making in code.

The If-Elif-Else Pattern

Conditionals in Python use if, elif (else if), and else. Think of them as road forks — your code checks a condition and goes down one path or another.

Indentation Rules

Python uses indentation (spaces at the start of a line) to define code blocks. Most languages use curly braces {}, but Python uses whitespace. Standard: 4 spaces per level. Mix tabs and spaces = error!

Common Mistake

Forgetting to indent after if: causes an IndentationError. Always indent the code block under a conditional by 4 spaces.

Loops

📖 12 min read🔄 Topic 6 of 13
Why This Matters

Loops let your code repeat actions automatically. Processing 1,000 data records? Loop. Checking each word in a document for sentiment? Loop. An AI agent retrying a failed API call? Loop. Repetition is everywhere in programming.

For Loops: "Do this for each item"

A for loop iterates over a sequence (list, range, string) and executes a block of code for each item. Think of it like a conveyor belt — each item gets the same treatment.

While Loops: "Keep going until..."

A while loop keeps running as long as a condition is True. Think of it like "keep stirring until the sauce thickens." Be careful — if the condition never becomes False, you get an infinite loop!

Loop Control

  • break — Exit the loop immediately ("stop the conveyor belt")
  • continue — Skip the rest of this iteration and go to the next one

Functions

📖 12 min read⚡ Topic 7 of 13
Why This Matters

Functions are reusable blocks of code. Instead of writing the same logic 10 times, you write it once as a function and call it 10 times. Every AI framework is built with functions — understanding them is non-negotiable.

The DRY Principle

Don't Repeat Yourself. If you find yourself copying and pasting code, that's a sign you need a function. Functions are like recipes — write the recipe once, cook it anytime.

Anatomy of a Function

  • def keyword — declares a function
  • Name — what you call it (use snake_case: calculate_area)
  • Parameters — inputs the function accepts
  • Body — the indented code that runs
  • return — the value the function sends back

Return vs Print

print() displays text on screen but the value is lost. return sends a value back to the caller so you can use it in further calculations. Think: print = showing, return = giving.

Data Structures

📖 14 min read📦 Topic 8 of 13
Why This Matters

Data structures are how you organize information in code. Choosing the right structure is like choosing the right container — you wouldn't store soup in a colander. In AI/ML, data structures determine how efficiently you can process information.

Lists — Ordered Collections

A list is an ordered, changeable collection. Think of it as a numbered shelf where you can add, remove, and rearrange items.

Dictionaries — Key-Value Pairs

A dictionary maps keys to values, like a real dictionary maps words to definitions. Use when you want to look up values by a name instead of a position number.

Tuples — Immutable Sequences

A tuple is like a list you can't change after creation. Use for data that shouldn't be modified, like coordinates (x, y) or RGB colors (255, 0, 0).

Sets — Unique Elements

A set is an unordered collection with no duplicates. Great for removing duplicates and checking membership quickly.

NumPy Essentials

📖 12 min read🔢 Topic 9 of 13
Why This Matters

NumPy is the backbone of scientific computing in Python. Every ML library (scikit-learn, TensorFlow, PyTorch) is built on NumPy. It makes math on large datasets 10-100x faster than plain Python lists. If you do ML, you use NumPy.

Why NumPy?

Python lists are flexible but slow for math. Adding two lists of 1 million numbers with a for loop takes seconds. NumPy does it in milliseconds because it runs optimized C code under the hood.

Key Concepts

  • ndarray — NumPy's core data type: a grid of numbers, all the same type
  • Vectorized operations — math on entire arrays at once, no loops needed
  • Broadcasting — NumPy automatically aligns arrays of different shapes
  • Shape — the dimensions of an array, e.g., (3, 4) = 3 rows, 4 columns

Why This Matters

APIs are how software talks to software. When an AI agent searches the web, checks the weather, or sends an email — it's calling an API. APIs are literally the "hands" that let agents interact with the real world.

What is an API?

API stands for Application Programming Interface. Think of it like a restaurant: you (the client) don't go into the kitchen (the server). You order through a waiter (the API) who brings back your food (the response).

HTTP Methods

MethodPurposeExample
GETRead/retrieve dataGet weather for a city
POSTCreate/send dataSend a message to ChatGPT
PUTUpdate existing dataUpdate user profile
DELETERemove dataDelete a record

Anatomy of an API Request

  • URL (endpoint) — where to send the request: https://api.example.com/data
  • Headers — metadata like authentication tokens, content type
  • Body — data you're sending (for POST/PUT)
  • Status Code — the response's success/failure indicator

Common Status Codes

CodeMeaningYou Should...
200OK — SuccessParse the response
201CreatedResource was created
400Bad RequestCheck your request format
401UnauthorizedCheck your API key
404Not FoundCheck the URL
429Rate LimitedSlow down, wait and retry
500Server ErrorNot your fault — retry later

API Security & Ethics

  • Authentication: Most APIs require a key. Keep it secret (use .env files).
  • Rate limiting: APIs limit how many requests you can make. Respect the limits.
  • Data privacy: Don't send personal data unless necessary.
  • Terms of service: Read them — some APIs prohibit certain uses.

Key Takeaways

  • APIs let software communicate: your code → API server → database → response
  • HTTP methods: GET (read), POST (create), PUT (update), DELETE (remove)
  • Status codes tell you what happened: 2xx = success, 4xx = your error, 5xx = server error
  • Always protect API keys and respect rate limits
